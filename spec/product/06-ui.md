# UI

## UI Type

Web dashboard — vanilla HTML + CSS + JavaScript (no framework). Served by FastAPI at `GET /`. Single-page layout with tab navigation. No build step required.

Separate from the generated blog site — the dashboard is the operator's control panel; `./output/` is the reader-facing blog.

---

## Views / Screens

### Tab 1 — Blog Settings

**Purpose:** Configure the blog's identity and generation settings.

**Key elements:**
- Blog name (text input)
- Tagline (text input, optional)
- Niche (text input — the primary topic focus)
- Themes (tag input — add/remove sub-themes)
- Posts per run (number input, 1–10)
- Schedule (cron input with human-readable preview, e.g. "Every Monday at 8am")
- Output directory (text input)

**Actions:**
- Save settings (PUT /blog)
- "Next scheduled run" indicator (shows when the next cron run will fire)

---

### Tab 2 — Writers

**Purpose:** Create and manage writer personas.

**Key elements:**
- List of all writers (name, bio preview, active/inactive badge)
- Add writer button → opens inline form

**Add/Edit writer form fields:**
- Name
- Persona prompt (large textarea with placeholder: "Describe this writer's voice, style, and areas of expertise...")
- Bio (textarea, 2–3 sentences)
- Avatar URL (optional)
- Active toggle

**Actions:**
- Add writer (POST /writers)
- Edit writer (PUT /writers/{id})
- Deactivate writer (DELETE /writers/{id}) — shows a confirmation prompt

---

### Tab 3 — Generate

**Purpose:** Trigger generation runs and see run history.

**Key elements:**
- "Generate Now" button with a post count selector (defaults to blog's posts_per_run)
- Active run status indicator (shows "Running..." with a spinner if a run is in progress)
- Run history table: date/time, trigger type, posts completed/requested, status, duration

**Actions:**
- Trigger manual run (POST /runs/trigger)
- Click a run row to see its posts (GET /runs/{id})

**Post list (within a run):**
- Title
- Writer name
- Topic
- Published timestamp
- Link to view the generated HTML post in a new tab

---

## Error States

- API errors: shown as a dismissible banner at the top of the page
- Run already in progress: "Generate Now" button is disabled with tooltip "A run is already in progress"
- No active writers: "Generate Now" is disabled with tooltip "Add at least one writer first"
- Blog not configured: Settings tab highlighted with a warning badge

## Tech Stack

- Vanilla HTML + CSS (no Tailwind, no React, no build step)
- Vanilla JavaScript `fetch()` for all API calls
- Served by FastAPI's `StaticFiles` or inline as a Jinja2 template
- Single file: `src/blogforge/dashboard/index.html` (self-contained)
