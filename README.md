# Data Analysis Agent

Upload a CSV (or connect a database) and ask questions about your data in plain English. Each dataset becomes a **Model Context Protocol (MCP) server** whose **tools, resources, and prompts** are LLM-generated by a sync pipeline and served over a JSON-RPC endpoint. A LangGraph ReAct agent answers questions by running read-only DuckDB SQL over the data; an LLM (via OpenRouter) does the reasoning.

> **All commands run from the repo root.**

---

## Quick Start

### 1. Install dependencies

```bash
# Install uv if you don't have it
curl -LsSf https://astral.sh/uv/install.sh | sh

# Install project dependencies
uv sync
```

### 2. Configure environment

```bash
cp .env.example .env
# Edit .env and set DATAANALYSIS_OPENROUTER_API_KEY to your OpenRouter key
# Leave it blank to run in stub mode (offline, no real AI)
```

### 3. Apply database migrations

```bash
uv run alembic upgrade head
uv run alembic current   # must print a revision hash — blank means migration failed
```

### 4. Run the app

```bash
uv run python -m data_analysis_agent
```

Open [http://localhost:8001](http://localhost:8001) in your browser.

## Features (v0.1)

- **MCP servers** — the primary entity. Each is backed 1:1 by a dataset (a named directory of Parquet files from uploaded CSVs, or an external PostgreSQL database in BETA). Its **tools, resources, and prompts** are LLM-generated by a 5-stage sync pipeline (title → JSONSchema → entities → GET-API tools → prompts), **versioned** and **soft-delete-only**.
- **MCP JSON-RPC API** — `POST /mcpserver/{id}` implements the MCP 2025-06-18 read surface: `tools/list` · `tools/call` · `resources/list` · `resources/read` · `prompts/list` · `prompts/get` (cursor pagination), for the UI and external MCP clients.
- **Natural Language Q&A** — type a question; a LangGraph ReAct agent calls a server (`{"tool","arguments"}`) and runs read-only DuckDB SQL (joins across a server's tables supported)
- **Sessions with memory** — each session keeps one MCP pool over its servers and remembers prior Q&A (durable across restarts), so follow-up questions have context
- **Query History** — review past questions and answers per session

## Stub Mode

If `DATAANALYSIS_OPENROUTER_API_KEY` is not set, the app runs in **stub mode**:
- A yellow banner appears on every page
- Answers are placeholder text (not real AI output)
- No API calls are made — safe for offline development

## Project Structure

```
src/data_analysis_agent/
├── api/          ← FastAPI routes: mcpserver.py, sessions.py, queries.py, home.py, health.py
├── config/       ← Settings (pydantic-settings)
├── db/           ← SQLAlchemy models + session
├── domain/       ← Pydantic domain models
├── graph/        ← LangGraph pipeline (state, nodes, edges, runner); SqliteSaver memory
├── llm/          ← OpenRouter + stub provider
├── tools/        ← CSV→Parquet ingestion, table naming
│   ├── connectors/  ← dataset connectors: uri, base (protocol+factory), parquet, postgres (BETA)
│   ├── sync/        ← 5-stage LLM capability generation (pipeline.py, stages.py)
│   └── mcp/         ← agent pool (pool.py) + per-server MCP (server.py) + JSON-RPC dispatch (dispatch.py)
└── templates/    ← Jinja2 HTML templates (home, session, mcpserver, base, error)

# internal datasets live under DATAANALYSIS_DATASETS_DIR/{slug(name)}/{table}.parquet

tests/
├── unit/         ← Pure unit tests (no DB, no network)
└── integration/  ← End-to-end pipeline + golden-path UI tests
```

## Stack

| Component | Choice |
|-----------|--------|
| Language | Python 3.12 |
| Web framework | FastAPI + uvicorn |
| UI | Jinja2 templates (React/Vite in Phase 4) |
| Agent | LangGraph (async nodes; one MCP pool + memory per session) |
| Primary entity | **MCP server** (1:1 with a dataset URI); children = tools/resources/prompts, generated by the sync pipeline (versioned, soft-delete) |
| Tool protocol | Model Context Protocol 2025-06-18 — `POST /mcpserver/{id}` JSON-RPC (DB-backed dispatcher) + official `mcp` SDK 1.28.0 (in-process FastMCP per server, generic `query` tool) for the agent |
| Datasets | URI-addressed metadata on the server: internal `parquet:///{name}` (dir of Parquet files) or external `postgresql://…` (BETA, flag-gated) |
| Query engine | DuckDB (read-only; `read_parquet` for internal, `ATTACH … READ_ONLY` for Postgres) |
| Agent memory | LangGraph `SqliteSaver` checkpointer (durable, per session) |
| LLM | via OpenRouter (default model `google/gemini-2.5-flash`) |
| Metadata DB | SQLite + SQLAlchemy 2.0 |
| Migrations | Alembic |
