# UI

## Technology

Jinja2 templates served by FastAPI. Minimal CSS (Bootstrap CDN). No JS framework.

## Screens

### 1. Home / New Project Form (`GET /`)

- Page title: "Construction Materials Sourcing Agent"
- If stub mode: visible banner at top: "⚠ Running in stub mode — results are simulated"
- Form fields:
  - Project Name (text input, required)
  - Materials table: rows of (Material Name, Quantity, Unit) with "Add row" button (plain JS)
  - Submit button: "Start Sourcing"
- On submit: POST to `/api/runs` (form-encoded or JSON), redirect to run status page

### 2. Run Status Page (`GET /runs/{run_id}`)

- Shows project name + run ID
- Status badge (pending / running / completed / failed)
- Auto-refreshes every 3 seconds (meta refresh or JS) until status is completed/failed
- If stub mode: visible banner
- On completed: shows link to "View Report"

### 3. Report Page (`GET /runs/{run_id}/report`)

- Shows project name + completion time
- If stub mode: visible banner
- Per-material section:
  - Material name + quantity + unit
  - Table of ranked suppliers: Rank | Supplier | Location | Price/Unit | Lead Time | Certifications | Score
  - Top recommendation highlighted (rank 1 row styled distinctly)
- Footer: "Prices and lead times are estimates. Verify with suppliers before ordering."

### 4. Error Page

- Shown when run status = failed
- Displays error_message from SourcingRun
- Link back to home
