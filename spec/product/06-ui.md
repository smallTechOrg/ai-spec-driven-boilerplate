# UI

## UI Type

Streamlit web app (single-page)

## Views / Screens

### Screen: Main / Itinerary

**Purpose:** User enters a city name and receives a travel + food itinerary.

**Key elements:**
- Page title: "Travel & Food Itinerary"
- Stub mode banner (yellow/warning) shown when `GOOGLE_API_KEY` is absent — **must be visible on every page load in stub mode**
- Text input: "Enter a city name" (placeholder: e.g. "Paris")
- Button: "Get Itinerary"
- Spinner shown while LLM call is in progress
- Results section — 3 place cards, each showing:
  - Place name (heading)
  - Description (body text)
  - Tips (smaller text, e.g. "🕐 Open 9am–6pm · Book tickets in advance")
- Dish section — 1 dish card showing:
  - Dish name (heading)
  - Description (body text)
- Error message shown if the LLM call fails

**Actions available:**
- Submit city name → triggers itinerary generation
- Clear / re-enter city name → re-run

## Error States

| Scenario | UI Behaviour |
|----------|--------------|
| Stub mode (no API key) | Yellow banner at top: "⚠ Running in stub mode — no API key detected. Output is hardcoded demo data." |
| LLM call fails | Red `st.error()` message: "Failed to generate itinerary. Please try again." |
| Empty city name submitted | Input validation: show `st.warning("Please enter a city name.")` |
| LLM returns malformed JSON | Parse error caught; show same red error as LLM failure |

## Tech Stack

Streamlit — Python-based reactive web framework, no JS required.
