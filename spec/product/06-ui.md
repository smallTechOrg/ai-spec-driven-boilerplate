# UI

## UI Type

Server-rendered web UI. FastAPI + Jinja2 templates. Minimal styling (plain HTML with inline CSS or a tiny stylesheet). No JavaScript framework required for v0.1.

## Views / Screens

### Screen: Upload Form (`GET /`)

**Purpose:** The user selects and submits a food photo.

**Key elements:**
- Page title: "Food Tracker"
- File input labelled "Upload a food photo"
- Submit button labelled "Analyse"
- Brief instruction text: "Supported formats: JPEG, PNG, HEIC. Max 10 MB."

**Actions available:**
- Select an image file and click Analyse — submits `POST /analyze`

---

### Screen: Result (`POST /analyze` response)

**Purpose:** Show the nutrition breakdown for the uploaded food.

**Key elements:**
- **Stub mode banner** (conditional): a clearly visible yellow/amber banner reading "DEMO MODE — results are simulated, not real AI output" when `provider == "stub"`
- Food name (large, prominent)
- Calories (kcal) displayed as a number
- Macros table: Protein (g) | Carbs (g) | Fat (g)
- "Analyse another photo" link back to `GET /`

**Actions available:**
- Click "Analyse another photo" — navigates to `GET /`

## Error States

| Condition | What the user sees |
|-----------|-------------------|
| No file selected | Form validation message: "Please select a photo" |
| File too large (> 10 MB) | Error page: "File exceeds the 10 MB limit. Please use a smaller image." |
| Gemini parse failure | Error page: "Could not parse nutrition data from this photo. Please try a clearer image." |
| Server error | Generic error page: "Something went wrong. Please try again." |

## Tech Stack

FastAPI + Jinja2 for server-rendered templates. No frontend framework. Plain HTML5 + minimal inline CSS.
