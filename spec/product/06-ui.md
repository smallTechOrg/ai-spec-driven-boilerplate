# UI

Server-rendered HTML via Jinja2. Plain CSS, no JS framework.

## Pages

### `/` — New Request

A form with one column of labeled inputs:
- Material (text)
- Quantity (text)
- Location (text)
- Budget (text, optional)
- Timeline (text, optional)
- Quality / preference criteria (textarea, optional)

Submit button: **"Find suppliers"**.

### `/runs/{run_id}` — Report

- Header: material + quantity + location echo of the request.
- Status pill: `running` / `completed` / `failed`.
- If completed: ordered list of recommendations
  - Rank, supplier name, score (0–100)
  - Price indication, lead time, location
  - Rationale paragraph
  - Source URL link (if any)
- If failed: red panel with `error_message`.

### `/runs` — Recent runs

Table: created_at, material, location, status, link to report.

## Stub-mode banner

The base template injects `{{ llm_provider }}` and `{{ search_provider }}`.
If either equals `stub`, the layout renders a yellow banner:

> **Demo / stub mode** — `LLM=stub`, `Search=tavily`. Set `SOURCING_GEMINI_API_KEY` and `SOURCING_TAVILY_API_KEY` in `.env` to use real providers.

The banner appears on every page.
