# Capability: Site Rendering

## What It Does

Writes the generated posts as plain HTML files to the output directory and regenerates the blog's `index.html` listing all posts, producing a browsable static site.

## Inputs

| Input | Type | Source | Required |
|-------|------|--------|---------|
| posts | list[Post] | Current run's completed posts | yes |
| blog | Blog | Blog config | yes |
| writers | dict[int, Writer] | Keyed by writer_id | yes |
| output_dir | str | Blog config | yes |

## Outputs

| Output | Type | Destination |
|--------|------|-------------|
| post HTML files | files | `{output_dir}/posts/{slug}.html` |
| index.html | file | `{output_dir}/index.html` |
| style.css | file | `{output_dir}/style.css` (written once; not overwritten if exists) |

## HTML Structure

### Individual post page (`posts/{slug}.html`)

```html
<!DOCTYPE html>
<html>
<head>
  <meta charset="utf-8">
  <title>{post.title} — {blog.name}</title>
  <link rel="stylesheet" href="../style.css">
</head>
<body>
  <header>
    <a href="../index.html">{blog.name}</a>
  </header>
  <article>
    <img src="../{cover_image_path}" alt="{post.title} cover">
    <h1>{post.title}</h1>
    <div class="byline">
      By {writer.name} · {published_at formatted as "April 23, 2026"}
    </div>
    {post.content_html}
    <div class="author-bio">
      <strong>{writer.name}</strong>
      <p>{writer.bio}</p>
    </div>
  </article>
  <footer>{blog.name} · Generated by BlogForge</footer>
</body>
</html>
```

### Index page (`index.html`)

Lists all posts (all time, newest first). Each entry shows: cover image (thumbnail), title (link to post page), writer name, date, and first 150 chars of post body as excerpt.

### style.css

A minimal, readable stylesheet: system font stack, max-width 720px centred, sensible heading sizes, responsive image sizing. Written once — never overwritten on subsequent runs (allows operator to customise).

## Business Rules

- Only posts with `status = completed` and a non-null `published_at` are included in the index
- Index is fully regenerated on every run (reflects all historical posts, not just the current run's)
- Post HTML files are never deleted; if a slug already exists, it is overwritten (re-renders the same post)
- `style.css` is written on first run only — if it already exists, it is left unchanged
- All paths in HTML are relative (site must be portable — no absolute URLs)
- Output directory is created if it does not exist

## Success Criteria

- [ ] After a run, `{output_dir}/index.html` lists all completed posts (newest first)
- [ ] Each post has a corresponding `{output_dir}/posts/{slug}.html` that is valid HTML
- [ ] All links in the generated HTML are relative and resolve correctly when opened from the output directory
- [ ] Opening `index.html` in a browser with no internet connection renders the full blog correctly
- [ ] `style.css` is not overwritten if it already exists
