# Agent

## Agent Architecture Pattern

**Chosen:** Graph (LangGraph) — multi-step pipeline with conditional edges. Two compiled graphs: `profile_graph` for upload profiling (no LLM), `qa_graph` for Q&A (LLM code generation → sandboxed execution → prose formatting).

---

## LLM Provider & Model

| Node | Provider | Model ID | Rationale |
|------|----------|----------|-----------|
| plan_and_code | Gemini | gemini-3.1-pro | Cost-efficient code generation from schema+stats |
| format_response | Gemini | gemini-3.1-pro | Prose formatting of execution result |
| profile_data | none | — | Pure pandas computation, zero LLM calls |
| execute_code | none | — | Sandboxed exec(), zero LLM calls |

---

## Tools / Capabilities

| Tool | Input | Output |
|------|-------|--------|
| profile_csv | file path | profile dict (schema, stats, quality flags) |
| execute_pandas | code string + DataFrames dict | result string + optional Plotly JSON |

## AgentState TypedDict

```python
from typing import TypedDict, Any

class AgentState(TypedDict, total=False):
    session_id: str              # UUID — identifies the session in SQLite
    action: str                  # "profile" | "answer" | "clarify" | "error"
    uploaded_files: list[dict]   # [{file_id, filename, path, profile_json, dataframe?}]
    current_question: str | None # The user's current question
    generated_code: str | None   # Python code generated by plan_and_code
    execution_result: str | None # str(result) from exec() sandbox
    chart_json: dict | None      # Plotly figure as JSON-serialisable dict (or None)
    answer: str | None           # Formatted prose answer from format_response
    error: str | None            # Error message if any node fails
    retry_count: int             # Number of reflect_and_retry attempts so far (max 2)
    quality_report: dict | None  # Quality inspection result (Phase 4); null if no issues found
    clean_actions: list[str]     # Human-readable log of auto-fixes applied (Phase 4)
```

---

## Nodes

### profile_data(state: AgentState) → AgentState
- Reads the CSV at `state["uploaded_files"][0]["path"]` with pandas
- Computes per-column: dtype, null_count, null_pct, sample_values (first 3 non-null stringified)
- Numeric columns: min, max, mean, std, p25, p50, p75
- Categorical columns: top-5 value_counts
- Quality flags: >20% nulls → WARNING; all-null → ERROR; duplicate rows → WARNING
- Sets `action="profile"` in returned state
- NO LLM call

### needs_clarification(state: AgentState) → AgentState
- Evaluates `current_question` against uploaded file profiles to detect ambiguity (e.g. multiple plausible columns, underspecified aggregation)
- Calls Gemini gemini-3.1-pro with schema + question; returns either a clarification question string or a signal to proceed
- If clarification needed: sets `action="clarify"`, sets `answer` to the clarification question, persists the assistant message to SQLite
- If not needed: passes state unchanged so `inspect_quality` runs next
- Added in Phase 3

### inspect_quality(state: AgentState) → AgentState
- Runs deterministically (NO LLM call) on every uploaded DataFrame in `uploaded_files`
- Per DataFrame inspections:
  - **Duplicates:** `df.duplicated().sum()` — if > 0, drop with `df.drop_duplicates()` (auto-fix)
  - **Numeric string columns:** for each object-dtype column, if `pd.to_numeric(col, errors='coerce').notna().all()` is True, coerce to float64 (auto-fix)
  - **Missing values:** for every column, compute null_count and null_pct; flag if null_pct > 0 (report only)
  - **Invalid dates:** for object-dtype columns whose name contains "date", "time", "dt", or "at" (case-insensitive), count values that fail `pd.to_datetime(errors='coerce')` (report only)
  - **Outliers:** for numeric columns, count values where `abs(x - mean) > 3 * std` (report only)
- Sets `quality_report` (dict or None) and `clean_actions` (list[str]) in returned state
- Stores cleaned DataFrames back into `uploaded_files` entries (in-memory); `execute_code` reads from there
- Privacy: `quality_report` contains ONLY column names, counts, percentages — never raw cell values
- On any pandas exception: logs warning, sets `quality_report = null`, continues with original DataFrames
- Added in Phase 4

### plan_and_code(state: AgentState) → AgentState
- Loads conversation history from SQLite (last 20 messages for session_id)
- Builds prompt: system instruction + schema (col names + dtypes, NO raw rows) + stats + history + question
- Calls Gemini gemini-3.1-pro to generate pandas code (stores answer in `result`, optional chart in `fig`)
- Operates on cleaned DataFrames already in state (post-`inspect_quality`) — does not re-read from disk
- Sets `generated_code` in returned state
- On Gemini error: sets `error`

### execute_code(state: AgentState) → AgentState
- Uses cleaned DataFrames already in AgentState `uploaded_files` (populated by `inspect_quality`; keyed by filename stem)
- Falls back to reading from disk if in-memory DataFrame is absent (e.g. direct invocation in tests)
- Runs `generated_code` via exec() in restricted namespace: `{dfs, pd, np, go, px}`
- Captures `result` → `execution_result` (str), `fig` → `chart_json` (Plotly JSON dict)
- On exception: sets `error` with traceback summary
- Timeout: 30 seconds

### reflect_and_retry(state: AgentState) → AgentState
- Triggered when `execute_code` sets `error` and `retry_count < 2`
- Calls Gemini gemini-3.1-pro with the failed code + error traceback; asks for a corrected version
- Increments `retry_count`, clears `error`, sets new `generated_code`
- If `retry_count >= 2`: routes to `handle_error` instead
- Added in Phase 3

### format_response(state: AgentState) → AgentState
- Calls Gemini gemini-3.1-pro with `execution_result` + `current_question`
- Returns plain prose answer for a non-technical reader
- Sets `answer`

### handle_error(state: AgentState) → AgentState
- Sets `action="error"`
- Sets `answer = f"I encountered an error: {state['error']}. Please try rephrasing your question."`

## Graph Edges

```
profile_graph:
  [START] → profile_data → [END]

qa_graph (Phases 1–4 cumulative):
  [START] → needs_clarification ─── clarify? ──→ [END]   (assistant message already persisted)
                                └── proceed ──→ inspect_quality
                                                    │
                                                    ▼
                                              plan_and_code ─── error? ──→ handle_error → [END]
                                                    │
                                                    └── ok ──→ execute_code ─── error + retry? ──→ reflect_and_retry
                                                                    │                                      │
                                                                    │ (retry loops back to execute_code)   │
                                                                    │◄─────────────────────────────────────┘
                                                                    ├── error + exhausted? ──→ handle_error → [END]
                                                                    └── ok ──→ format_response → [END]
```

Conditional edge functions:
- `after_clarification(state)`: returns `END` if `state.get("action") == "clarify"`, else `"inspect_quality"`
- `after_plan(state)`: returns `"handle_error"` if `state.get("error")`, else `"execute_code"`
- `after_execute(state)`:
  - if `state.get("error")` and `state.get("retry_count", 0) < 2`: returns `"reflect_and_retry"`
  - if `state.get("error")` and `state.get("retry_count", 0) >= 2`: returns `"handle_error"`
  - else: returns `"format_response"`
- `after_reflect(state)`: always returns `"execute_code"` (retry loop)

## Runner Entry Points

```python
def run_profile(session_id: str, file_id: str, filename: str, file_path: str) -> dict:
    """Profile a freshly uploaded file. Returns profile dict."""
    initial: AgentState = {
        "session_id": session_id,
        "action": "profile",
        "uploaded_files": [{"file_id": file_id, "filename": filename, "path": file_path}],
    }
    final = profile_graph.invoke(initial)
    return final["uploaded_files"][0]["profile_json"]

def run_question(session_id: str, question: str, uploaded_files: list[dict]) -> dict:
    """Answer a natural-language question. Returns {answer, chart_json, quality_report}."""
    initial: AgentState = {
        "session_id": session_id,
        "action": "answer",
        "current_question": question,
        "uploaded_files": uploaded_files,
        "retry_count": 0,
        "quality_report": None,
        "clean_actions": [],
    }
    final = qa_graph.invoke(initial)
    return {
        "answer": final.get("answer"),
        "chart_json": final.get("chart_json"),
        "quality_report": final.get("quality_report"),
    }
```

## Memory Strategy

- Within a session: conversation history in SQLite `messages` table
- Each Q&A turn loads last 20 messages from DB for LLM prompt context (10 user + 10 assistant)
- No cross-session memory; all data deleted on session end
- AgentState is ephemeral per invocation — no LangGraph checkpointing in Phase 1

---

## Error Handling

| Failure | Response |
|---------|----------|
| Gemini API error | route to handle_error → return error message to user |
| CSV read error | route to handle_error |
| exec() exception | catch, set error, route to handle_error |
| exec() timeout (> 30s) | kill, set error, route to handle_error |
| Missing session | raise 404 HTTP error at API layer before invoking graph |

---

## Observability

- structlog structured logging on every node entry/exit: session_id, node name, duration_ms
- Log Gemini token usage (prompt_tokens, completion_tokens) per call to stdout
- LangSmith tracing: optional — enabled when LANGCHAIN_TRACING_V2=true + LANGCHAIN_API_KEY set in .env

## Graph Assembly (pseudocode)

```python
from langgraph.graph import StateGraph, END
from src.graph.state import AgentState
from src.graph.nodes import (
    profile_data, needs_clarification, inspect_quality,
    plan_and_code, execute_code, reflect_and_retry,
    format_response, handle_error,
)
from src.graph.edges import after_clarification, after_plan, after_execute, after_reflect

# Profile graph (upload path — no LLM, no quality check)
pb = StateGraph(AgentState)
pb.add_node("profile_data", profile_data)
pb.set_entry_point("profile_data")
pb.add_edge("profile_data", END)
profile_graph = pb.compile()

# QA graph (question path — Phases 1–4 cumulative)
qb = StateGraph(AgentState)
qb.add_node("needs_clarification", needs_clarification)   # Phase 3
qb.add_node("inspect_quality", inspect_quality)           # Phase 4
qb.add_node("plan_and_code", plan_and_code)
qb.add_node("execute_code", execute_code)
qb.add_node("reflect_and_retry", reflect_and_retry)       # Phase 3
qb.add_node("format_response", format_response)
qb.add_node("handle_error", handle_error)

qb.set_entry_point("needs_clarification")
qb.add_conditional_edges(
    "needs_clarification", after_clarification,
    {"inspect_quality": "inspect_quality", END: END}
)
qb.add_edge("inspect_quality", "plan_and_code")
qb.add_conditional_edges(
    "plan_and_code", after_plan,
    {"execute_code": "execute_code", "handle_error": "handle_error"}
)
qb.add_conditional_edges(
    "execute_code", after_execute,
    {"reflect_and_retry": "reflect_and_retry", "format_response": "format_response", "handle_error": "handle_error"}
)
qb.add_conditional_edges(
    "reflect_and_retry", after_reflect,
    {"execute_code": "execute_code"}
)
qb.add_edge("format_response", END)
qb.add_edge("handle_error", END)
qa_graph = qb.compile()
```
