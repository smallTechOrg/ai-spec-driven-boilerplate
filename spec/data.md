# Data Model

## Storage Technologies

Two distinct stores:

1. **In-memory session store** — a process-level Python dict: `SESSION_STORE: dict[str, pd.DataFrame]`. Holds the uploaded CSV data for the lifetime of the server process. Keys are `session_id` UUIDs. Values are pandas DataFrames. Never written to disk, never sent externally.

2. **SQLite** (`data/agent.db`) — stores session metadata and conversation run history only. Created by Alembic migrations. Contains no CSV data.

---

## In-Memory Session Store

**Module:** `src/sessions/store.py`

```python
SESSION_STORE: dict[str, pd.DataFrame] = {}
```

| Key | Value |
|-----|-------|
| `session_id` (UUID string) | `pd.DataFrame` — the full uploaded CSV |

**Lifecycle:**
- Created on `POST /sessions` success.
- Read on every `POST /sessions/{session_id}/questions` call.
- Lost on process restart (session must be re-uploaded after restart).
- Never explicitly deleted by the application.

---

## Entities (SQLite, Alembic-managed)

### Entity: `Session`

Represents a single CSV file upload session. Created when the user uploads a CSV and the DataFrame is successfully stored in `SESSION_STORE`.

SQLAlchemy model: `src/db/models.py` → `class Session`
Table name: `sessions`

| Field | SQLAlchemy Type | Nullable | Description |
|-------|-----------------|----------|-------------|
| `id` | `Text` PK | No | UUID string, generated by `uuid4()`. Returned to client as `session_id`. |
| `original_filename` | `Text` | No | The filename as uploaded, stored for display purposes. |
| `row_count` | `Integer` | No | Number of data rows (excluding header). |
| `col_count` | `Integer` | No | Number of columns. |
| `columns_json` | `Text` | No | JSON string: `[{"name": "col", "dtype": "float64"}, ...]`. Stored so the schema can be returned without re-inspecting the DataFrame. |
| `created_at` | `TIMESTAMP(timezone=True)` | No | UTC timestamp, set at insert, never updated. |

---

### Entity: `ConversationRun`

Represents one question-answer exchange. Created by `POST /sessions/{session_id}/questions` when the pipeline starts.

SQLAlchemy model: `src/db/models.py` → `class ConversationRun`
Table name: `conversation_runs`

| Field | SQLAlchemy Type | Nullable | Description |
|-------|-----------------|----------|-------------|
| `id` | `Text` PK | No | UUID string. Returned to client as `run_id`. |
| `session_id` | `Text` FK → `sessions.id` | No | The session this run belongs to. |
| `question` | `Text` | No | The natural-language question as submitted. |
| `answer` | `Text` | Yes | Plain-English answer from Gemini. NULL if pipeline failed before `answer_question`. |
| `chart_data` | `Text` | Yes | JSON object: `{"chart_base64": "<base64 PNG>", "chart_type": "bar"}`. NULL in Phase 1; populated in Phase 2. |
| `executed_code` | `Text` | Yes | The Python code that was executed. NULL in Phase 1; populated in Phase 2. |
| `node_trace` | `Text` | No | JSON array of node names that ran, in order. e.g. `["parse_csv", "answer_question", "finalize"]`. |
| `status` | `Text` | No | `"pending"` → `"completed"` or `"failed"`. |
| `error` | `Text` | Yes | Human-readable error message if `status = "failed"`. NULL otherwise. |
| `created_at` | `TIMESTAMP(timezone=True)` | No | UTC timestamp, set at insert. |
| `updated_at` | `TIMESTAMP(timezone=True)` | No | UTC timestamp, set at insert and updated on status change. |

---

## Relationships

```
Session (1) ──── (many) ConversationRun
   via ConversationRun.session_id → Session.id
```

Each `Session` has zero or more `ConversationRun` records. Each `ConversationRun` belongs to exactly one `Session`.

---

## Data Lifecycle

| Event | Action |
|-------|--------|
| `POST /sessions` succeeds | DataFrame stored in `SESSION_STORE`; `Session` row inserted in SQLite. |
| `POST /sessions/{id}/questions` called | `ConversationRun` row inserted with `status="pending"`. |
| Pipeline succeeds | `ConversationRun` updated: `status="completed"`, `answer`, `node_trace`, optionally `chart_data` + `executed_code` populated. |
| Pipeline fails | `ConversationRun` updated: `status="failed"`, `error` populated. |
| Process restart | `SESSION_STORE` is cleared; DataFrames are lost. `Session` metadata remains in SQLite but any new question for the old `session_id` will fail with "Session not found". |
| Manual cleanup | SQLite rows can be deleted manually. No TTL or automatic cleanup in scope. |

---

## Sensitive Data

- `ConversationRun.question` may contain business-sensitive data. Stored unencrypted in SQLite.
- The CSV data itself is held in-memory and is never written to disk or logged. Users are responsible for not uploading PII-containing CSVs.
- No API keys or credentials are stored in the database. They live exclusively in `.env`.
