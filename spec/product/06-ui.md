# UI

## UI Type

Web dashboard — vanilla HTML + CSS + JavaScript (no framework). Served by FastAPI at `GET /`. Single-page layout with 4-tab navigation. No build step required.

The dashboard IS the primary interface for viewing generated content. There is no separate reader-facing site in the initial build (static HTML export is a future phase — see `spec/product/capabilities/04-site-rendering.md`).

---

## Views / Screens

### Tab 1 — Blog Settings

**Purpose:** Configure the blog's identity and generation settings.

**Key elements:**
- Blog name (text input)
- Tagline (text input, optional)
- Niche (text input — the primary topic focus)
- Themes (tag input — add/remove sub-themes)
- Posts per run (number input, 1–10)
- Schedule (cron input with human-readable preview, e.g. "Every Monday at 8am")

**Actions:**
- Save settings (`PUT /blog`)
- "Next scheduled run" indicator

---

### Tab 2 — Writers

**Purpose:** Create and manage writer personas.

**Key elements:**
- List of all writers (name, bio preview, active/inactive badge)
- Add writer button → opens inline form

**Add/Edit writer form fields:**
- Name
- Persona prompt (large textarea)
- Bio (textarea, 2–3 sentences)
- Avatar URL (optional)
- Active toggle

**Actions:**
- Add writer (`POST /writers`)
- Edit writer (`PUT /writers/{id}`)
- Deactivate writer (`DELETE /writers/{id}`) with confirmation prompt

---

### Tab 3 — Generate

**Purpose:** Trigger generation runs and see run history.

**Key elements:**
- "Generate Now" button with post count selector (defaults to blog's posts_per_run)
- Active run status indicator (spinner + "Running..." if a run is in progress)
- Run history table: date/time, trigger type, posts completed/requested, status, duration

**Actions:**
- Trigger manual run (`POST /runs/trigger`)
- Click a run row to jump to the Library tab filtered to that run's posts

**Disabled states:**
- "Generate Now" disabled if a run is already in progress (tooltip: "A run is already in progress")
- "Generate Now" disabled if no active writers configured (tooltip: "Add at least one writer first")
- Settings tab highlighted with warning badge if blog niche is empty

---

### Tab 4 — Library (Content Library)

**Purpose:** Browse all generated posts. This is the primary output view.

**Key elements:**
- Filterable/searchable list of all posts (newest first)
- Filter by writer, date range
- Each post card: title, writer name + avatar, topic, date, word count, cover image thumbnail
- Click a post → opens full post detail view (modal or expanded panel)

**Post detail view:**
- Cover image (full width)
- Title
- Writer byline (name + bio)
- Full post body (rendered Markdown → HTML, displayed inline)
- Topic tag, date, word count
- Copy Markdown button

**Actions:**
- Filter posts (`GET /posts?writer_id=&limit=&offset=`)
- View full post content
- Copy post Markdown to clipboard

---

## Error States

- API errors: dismissible banner at the top of the page
- Run in progress: Generate button disabled with tooltip
- No active writers: Generate button disabled with tooltip
- Blog not configured: Settings tab shows warning badge
- Empty library: Library tab shows "No posts yet — trigger your first run"

## Tech Stack

- Vanilla HTML + CSS (no Tailwind, no React, no build step)
- Vanilla JavaScript `fetch()` for all API calls
- Served by FastAPI (`StaticFiles` or inline Jinja2 template)
- Single file: `src/blogforge/dashboard/index.html` (self-contained)
