# UI

## UI Type

Server-rendered HTML (FastAPI + Jinja2). Minimal styling via a tiny inline stylesheet.

## Screens

### Dashboard `/`
Links to Voices, Writers, Articles. Shows counts and last 5 articles.

### Voices `/voices`, `/voices/new`
List + form (name, description, guidelines markdown).

### Writers `/writers`, `/writers/new`
List + form (name, persona markdown, voice dropdown).

### Articles `/articles`, `/articles/new`, `/articles/{id}`
- `new`: writer dropdown + topic input + optional notes → submit
- `/articles`: newest-first list
- `/articles/{id}`: title + body (rendered markdown)

## Error States

Errors render as a red banner at the top of the current page. LLM failures show the `AgentRun.error_message` and keep the user on `/articles/new`.

## Tech Stack

Jinja2 templates, zero JS, system font, ~80-line inline CSS.
