# UI

## UI Type

Minimal server-rendered web UI (FastAPI + Jinja2 + Starlette `TemplateResponse` v1.0 signature). No JavaScript framework.

## Global Layout

All pages extend `base.html`:
- Top nav: Home · Runs · Leads · Download CSV
- **Stub-mode banner** rendered on every page when `llm_provider=stub` — "Stub LLM — offline demo output, not real AI". Required by non-negotiable rule 7.

## Views / Screens

### Screen: Home (`/`)

**Purpose:** Trigger a new run.

**Key elements:**
- Form with selects: `country` (EU_COUNTRIES), `industry` (free text), `size_band` (1-10/11-50/51-200/201-500)
- Submit button → `POST /runs`

### Screen: Runs (`/runs`)

**Purpose:** See history of pipeline runs.

**Key elements:**
- Table: created_at, filters, status, lead count, error_message (if failed)

### Screen: Leads (`/leads`)

**Purpose:** Browse the ranked leads across all runs; filter.

**Key elements:**
- Filter bar: country, industry, size_band, min_score
- Table: name (link to website), country, industry, size_band, hq_city, score, rationale
- "Download CSV" link → `/leads.csv` (preserving current filters)

## Error States

- Form validation errors render inline above the form.
- Pipeline failures render on the Runs page with the `error_message` column populated.
- Unhandled exceptions render a simple error page with a "back to home" link.

## Tech Stack

Jinja2 templates, vanilla HTML + a single CSS file at `api/static/style.css`. No client-side JS.
