# Capability: Natural Language to SQL

## What It Does

Translates a user's plain-English question into one or more DuckDB SQL queries using Gemini tool-use, executes those queries, and returns the raw result set to the agent for response synthesis.

## Inputs

| Input | Type | Source | Required |
|-------|------|--------|----------|
| session_id | UUID string | Chat API request | Yes |
| user_message | string | Chat API request | Yes |
| relevant_schemas | JSON array of `{table_name, columns: [{name, dtype}]}` | Context Builder (see [Capability: Token Economy](05_token_economy.md)) | Yes |
| conversation_history | array of `{role, content}` | SQLite `conversation_turns`; may be a summary + recent turns | Yes |

## Outputs

| Output | Type | Destination |
|--------|------|-------------|
| generated_sql | string | Audit log; passed to Response Synthesiser |
| result_set | JSON array of row objects | Passed to Response Synthesiser |
| row_count_returned | integer | Audit log |
| datasets_touched | array of table_name strings | Audit log |
| tool_call_log | JSON array of tool calls and results | Internal; not persisted separately |
| error_message | string or null | Passed to Response Synthesiser if SQL fails |

## External Calls

| System | Operation | On Failure |
|--------|-----------|------------|
| Gemini API | `generate_content` with tool definitions; may invoke multiple tool-use rounds | Return HTTP 502 to client with user-visible message; log full Gemini error |
| DuckDB | Execute SQL returned by Gemini via `execute_sql` tool | Return structured SQL error to Gemini as tool result; Gemini may retry with corrected SQL |
| SQLite (via SQLAlchemy) | Read `datasets` rows for schema context (via Context Builder) | Fatal: return HTTP 500 |

## Business Rules

- The agent never executes SQL that has not been generated by Gemini in the current turn; no user-supplied raw SQL is executed.
- Before execution, every SQL statement is checked against a destructive-SQL guard (see [Capability: Senior Analyst Workflow](04_senior_analyst.md)). Statements containing `DROP`, `DELETE`, `TRUNCATE`, or `ALTER` are rejected; the refusal is returned to Gemini as a tool error, and Gemini must produce a safe alternative or explain the refusal to the user.
- Only SELECT statements (and CTEs using SELECT) are permitted.
- Gemini may call `execute_sql` multiple times per turn (sub-queries, follow-up verification). Each call is recorded individually in the tool call log.
- If Gemini returns a final text response without any tool calls (e.g., a clarifying question), the capability returns immediately with no SQL executed and `generated_sql = null`.
- The maximum number of tool-call rounds per turn is 10. If this limit is reached, the agent returns a "could not complete query" error to the user.
- Result sets larger than 1,000 rows are truncated to 1,000 rows for the response; the user is informed of the truncation in the narrative.

> **Assumed:** Gemini is given the full result set (up to 1,000 rows) as the tool result; the response synthesiser then formats a markdown table from that. If the result set is very large (>50 rows), the synthesiser shows the first 20 rows and summarises the rest.

## Success Criteria

- [ ] A natural-language question about an uploaded dataset causes Gemini to call `execute_sql` with a syntactically valid DuckDB SQL SELECT statement.
- [ ] The SQL executes without error and the result set is returned to the caller.
- [ ] A question that requires joining two tables results in SQL that references both table names present in the DuckDB registry.
- [ ] A question with no matching table causes Gemini to surface a "no relevant table found" message rather than hallucinating a table name.
- [ ] A question that would require destructive SQL results in the guard rejecting the tool call and the user receiving a clear refusal message.
- [ ] A zero-row result does not cause an error; the user receives a "no results found" narrative.
- [ ] The `datasets_touched` field in the audit log correctly lists all tables referenced in the executed SQL.
- [ ] If Gemini returns a clarifying question (no tool call), `generated_sql` is null and no audit log entry for SQL execution is written.
